Guía técnica · v2 · Definitiva

Checklist del entorno para desarrollar y desplegar Java EE sobre WildFly

Una guía consolidada que incorpora todos los aprendizajes y errores corregidos durante la puesta a punto inicial del proyecto (Java+MySQL+JBOSS EE+Maven) sobre Windows 11 Pro.

SO: Windows 11 Pro 22H2+ Java: 21 LTS Servidor: WildFly 39.0.1.Final
00

Antes de empezar

Lecciones aprendidas la primera vuelta. Leelas antes de tocar nada, te ahorra de 1 a 4 horas de troubleshooting.

!
Versión exacta de WildFly Tenés que descargar WildFly 39.0.1.Final. La versión 40 (Beta o Final) es incompatible con Spring Boot 3.5 porque implementa Jakarta EE 11, mientras que el proyecto usa Jakarta EE 10. Si bajás la 40 vas a tener errores de tipo "EntityManagerFactory interface seems to conflict".
!
No copies la sección de proxy si no la necesitás Si no estás detrás de un proxy corporativo, saltá la Sección 8.2 completa. Configurar un proxy ficticio en ~/.m2/settings.xml rompe todas las builds de Maven con "Host desconocido".
i
PowerShell, no CMD Todos los comandos están pensados para PowerShell. Si abrís CMD, varios no van a funcionar. Usá Windows Terminal si lo tenés, o el ícono de PowerShell del menú inicio.
i
Cerrá y reabrí la terminal después de cada instalación Las variables de entorno (PATH, JAVA_HOME, WILDFLY_HOME) solo se refrescan en sesiones nuevas. Esta es la causa #1 de "el comando no se reconoce" después de instalar algo.

Convenciones de la guía

  • Pasos con casilla son verificaciones que tenés que poder marcar antes de seguir.
  • Bloques negros son código para pegar tal cual en PowerShell.
  • Bloques punteados muestran la salida esperada del comando anterior.
  • Cajas de color destacan información crítica (rojo), advertencias (amarillo) o tips (azul).
01

Sistema operativo y herramientas base

Configuraciones de Windows que tenés que aplicar antes de instalar nada. Si las dejás para después, varias instalaciones se rompen y hay que rehacerlas.

1 Verificar versión de Windows 
winver

Tiene que ser Windows 11 Pro, versión 22H2 o superior. Si aparece Windows 10 o una versión 21H1/21H2, actualizá antes de seguir.

1.1 Confirmé Windows 11 Pro 22H2 o superior
2 Habilitar PowerShell para ejecutar scripts

Por defecto Windows bloquea la ejecución de scripts .ps1. Como npm, Claude Code y varias herramientas se invocan así, hay que habilitarlo primero. Abrí PowerShell como Administrador:

Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

Confirmá con S (o Y en Windows en inglés). Verificá:

Get-ExecutionPolicy -List
Salida esperada (línea de CurrentUser)CurrentUser RemoteSigned
1.2 ExecutionPolicy CurrentUser en RemoteSigned
3 Habilitar paths largos

Windows trunca paths a 260 caracteres por defecto, lo que rompe Maven y npm con dependencias profundas. Como administrador:

New-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\FileSystem" `
  -Name "LongPathsEnabled" -Value 1 -PropertyType DWORD -Force
1.3 LongPathsEnabled = 1 en el registro
4 UTF-8 a nivel sistema

Tu proyecto está en español. Sin esto, Maven y los archivos con acentos se comportan raro.

  1. Panel de Control → Reloj y región → Región.
  2. Tab "Administrativo" → "Cambiar configuración regional del sistema".
  3. Marcar la casilla "Beta: Usar Unicode UTF-8 para compatibilidad de idiomas".
  4. Reiniciar la máquina.

Después del reinicio verificá:

[System.Text.Encoding]::Default.WebName
Salida esperadautf-8
1.4 Codificación del sistema en UTF-8
5 Espacio en disco 
Get-PSDrive C | Select-Object Used,Free

Necesitás al menos 20 GB libres. Maven, MySQL, WildFly y node_modules ocupan bastante.

1.5 Al menos 20 GB libres en C:
6 Git for Windows 
winget install --id Git.Git -e --source winget

Cerrá y reabrí PowerShell. Después configurá:

git --version
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"
git config --global core.autocrlf true
git config --global init.defaultBranch main
Git for Windows
gitforwindows.org · instalador alternativo

3 Validar que puede bajar de Maven Central
i
Por qué no usamos archetype:generate En Maven 3.9 ese comando se comporta distinto a versiones anteriores y suele fallar con "no POM in this directory". Validamos con un POM mínimo, que es más confiable.

Creá una carpeta de prueba:

mkdir $env:USERPROFILE\test-maven
cd $env:USERPROFILE\test-maven

Generá un pom.xml mínimo:

$pom = @'
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0">
  <modelVersion>4.0.0</modelVersion>
  <groupId>com.test</groupId>
  <artifactId>hello</artifactId>
  <version>1.0.0</version>
  <properties>
    <maven.compiler.release>21</maven.compiler.release>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
  </properties>
  <dependencies>
    <dependency>
      <groupId>org.junit.jupiter</groupId>
      <artifactId>junit-jupiter</artifactId>
      <version>5.11.3</version>
      <scope>test</scope>
    </dependency>
  </dependencies>
</project>
'@
[System.IO.File]::WriteAllText("$env:USERPROFILE\test-maven\pom.xml", $pom, [System.Text.UTF8Encoding]::new($false))

Probar:

mvn dependency:resolve

Tiene que terminar en BUILD SUCCESS y haber bajado JUnit y sus transitivas. Limpiá:

cd $env:USERPROFILE
Remove-Item -Recurse -Force test-maven
!
Si te falla con "Host desconocido (proxy.empresa.com)" Hay un proxy ficticio configurado en tu settings.xml. Borralo o desactivalo:
Remove-Item $env:USERPROFILE\.m2\settings.xml

El servidor de aplicaciones donde se va a desplegar el WAR. La versión exacta importa: tiene que ser 39.0.1.Final, no la 40.

!
No descargues WildFly 40 (Beta o Final) WildFly 40 implementa Jakarta EE 11, mientras que Spring Boot 3.5 está pensado para Jakarta EE 10. Si bajás la 40 vas a tener errores tipo "EntityManagerFactory interface seems to conflict with Spring's EntityManagerFactoryInfo mixin" al desplegar. Spring Boot 4 será compatible con EE 11 más adelante.

Driver MySQL como módulo en WildFly

Para que WildFly pueda exponer un datasource JNDI conectado a MySQL, hay que registrar el driver como módulo.

mysql-connector-j 9.1.0 · JAR directo
Maven Central · ~2.5 MB
1 Descargar el driver 
mkdir C:\Tools\drivers -Force
Invoke-WebRequest -Uri "https://repo1.maven.org/maven2/com/mysql/mysql-connector-j/9.1.0/mysql-connector-j-9.1.0.jar" `
  -OutFile C:\Tools\drivers\mysql-connector-j-9.1.0.jar
2 Crear el módulo 
$moduleDir = "$env:WILDFLY_HOME\modules\com\mysql\main"
New-Item -ItemType Directory -Force -Path $moduleDir | Out-Null
Copy-Item C:\Tools\drivers\mysql-connector-j-9.1.0.jar $moduleDir\
!
Crear el module.xml en UTF-8 sin BOM Si lo creás con Notepad común, agrega un BOM que rompe el parseo XML de WildFly. Usá el comando de abajo, que escribe UTF-8 sin BOM. 
$moduleXml = @'
<?xml version="1.0" encoding="UTF-8"?>
<module xmlns="urn:jboss:module:1.9" name="com.mysql">
  <resources>
    <resource-root path="mysql-connector-j-9.1.0.jar"/>
  </resources>
  <dependencies>
    <module name="java.se"/>
    <module name="jakarta.transaction.api"/>
  </dependencies>
</module>
'@

[System.IO.File]::WriteAllText(
  "$env:WILDFLY_HOME\modules\com\mysql\main\module.xml",
  $moduleXml,
  [System.Text.UTF8Encoding]::new($false)
)
7.1 Módulo com.mysql creado con JAR y module.xml (UTF-8 sin BOM)
3 Registrar driver y datasource

Con WildFly corriendo, abrí otra terminal:

& "$env:WILDFLY_HOME\bin\jboss-cli.bat" --connect

Dentro del prompt CLI:

/subsystem=datasources/jdbc-driver=mysql:add(driver-name=mysql,driver-module-name=com.mysql,driver-class-name=com.mysql.cj.jdbc.Driver)
data-source add --name=TestdbDS --jndi-name=java:jboss/datasources/TestdbDS --driver-name=mysql --connection-url=jdbc:mysql://localhost:3306/testdb?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=America/Argentina/Buenos_Aires --user-name=testdb --password=testdb --min-pool-size=5 --max-pool-size=50 --enabled=true --use-java-context=true
/subsystem=datasources/data-source=TestdbDS:test-connection-in-pool
Esperado{"outcome" => "success", "result" => [true]}

Salí del CLI con exit.

Troubleshooting · errores comunes

Catálogo de errores que aparecen seguido durante la instalación, con su causa y la solución.

"No se puede cargar el archivo npm.ps1 porque la ejecución de scripts está deshabilitada"

Causa: Windows bloquea scripts .ps1 por defecto.

Solución: PowerShell como admin → Set-ExecutionPolicy -Scope CurrentUser RemoteSigned → confirmar con S → cerrar y reabrir terminal.

"mysql: El término 'mysql' no se reconoce"

Causa: El instalador de MySQL no agrega bin al PATH del sistema.

Solución: Sección 5 paso 2 de esta guía. Agregar manualmente C:\Program Files\MySQL\MySQL Server 8.4\bin al PATH del sistema y reabrir terminal.

Maven: "Host desconocido (proxy.empresa.com)"

Causa: Hay un proxy ficticio configurado en ~/.m2/settings.xml.

Solución: Remove-Item $env:USERPROFILE\.m2\settings.xml. Si necesitás proxy real, configuralo con los datos verdaderos de tu organización.

Maven: "no POM in this directory" con archetype:generate

Causa: Maven 3.9.x cambió cómo invoca plugins sin POM en el directorio.

Solución: No uses archetype:generate para validar. Usá un POM mínimo y mvn dependency:resolve (sección 3 paso 3 de esta guía).

WildFly: "LoggerFactory is not a Logback LoggerContext but Logback is on the classpath"

Causa: Conflicto entre el Logback que viene en el WAR de Spring Boot y el subsistema de logging de WildFly.

Solución: Agregar src/main/webapp/WEB-INF/jboss-deployment-structure.xml excluyendo los módulos de logging (sección 9 paso 3).

Spring Boot: "Failed to determine a suitable driver class"

Causa: Spring Boot auto-configura un DataSource pero no encuentra config de conexión.

Solución: Crear application.properties con spring.datasource.url, username, password, driver-class-name y deshabilitar Flyway si no hay migraciones (sección 9 paso 2).

WildFly: "EntityManagerFactory interface [interface org.hibernate.SessionFactory] seems to conflict"

Causa: Estás corriendo WildFly 40 (Jakarta EE 11) con Spring Boot 3.5 (Jakarta EE 10).

Solución: Verificá la versión con Get-Content "$env:WILDFLY_HOME\README.txt" -TotalCount 5. Si dice 40, descargá WildFly 39.0.1.Final y reapuntá WILDFLY_HOME.

WildFly: el zip descargado pesa unos pocos KB

Causa: La URL falló y bajaste un HTML de error en vez del binario.

Solución: Ir manualmente a GitHub Releases y descargar el zip desde la sección "Assets" del navegador. El archivo debe pesar ~210 MB.

"JAVA_HOME no está definido" al arrancar WildFly o Maven

Causa: La variable se seteó pero la terminal no la refrescó.

Solución: Cerrar TODAS las ventanas de PowerShell y abrir una nueva. Si tampoco lo toma, reiniciar la sesión de Windows.

where.exe java muestra varias rutas

Causa: Hay un Java viejo (típicamente Oracle) en el PATH antes que Temurin 21.

Solución: Win+R → sysdm.cpl → Variables de entorno → editar PATH del sistema → eliminar entradas Oracle\Java\javapath, asegurarte que %JAVA_HOME%\bin queda arriba.

MySQL: "Public Key Retrieval is not allowed"

Causa: MySQL 8 usa caching_sha2_password por defecto y no envía la clave pública si la conexión no es SSL.

Solución: Agregar &allowPublicKeyRetrieval=true al connection-url, tanto en application.properties como en el datasource de WildFly.

Maven build extremadamente lento

Causa: Antivirus escaneando cada JAR descargado al .m2.

Solución: Aplicar exclusiones del antivirus de la sección 8.1 y reiniciar la terminal. La primera build siempre es lenta (baja todo Spring Boot); las siguientes son rápidas.
